Chat Control 2.0 reprend, Signal menace de quitter l'Europe

Le trilogue europeen sur le reglement CSAR — la version permanente de ce que tout le monde appelle « Chat Control » — a repris le 4 mai a Bruxelles. La Commission, le Parlement et le Conseil cherchent un accord politique pour juillet. Signal, par la voix de Meredith Whittaker, a redit cette semaine ce qu'elle dit depuis 2022 : si la version du Conseil — qui prevoit un client-side scanning obligatoire — l'emporte, Signal quitte l'Union europeenne plutot que d'introduire le mecanisme dans son protocole.

Le contexte n'est pas mince. La derogation temporaire qui autorisait depuis 2021 Google, Meta, Microsoft et TikTok a scanner volontairement les messages prives au nom de la lutte contre les contenus pedopornographiques a expire le 3 avril, apres le rejet par le Parlement europeen (311 voix contre 228) de toute extension. Pendant cinq semaines, les plateformes operant en Europe se sont retrouvees sans base legale pour ce scan de masse — un precedent rare en matiere de privacy europeenne. Le reglement permanent qui revient maintenant ressemble a la version 2.0 du meme probleme : sous le nouveau vocabulaire de « risk mitigation measures » et de « voluntary activities », le mecanisme de scan client-side reste sur la table, et la verification d'age generalisee glisse vers le statut de defaut.

C'est ici que la frontiere surveillance / open source devient floue. Briser le chiffrement de bout en bout dans Signal n'est pas une operation cote serveur : c'est une operation cote code. Pour un protocole open source, c'est un fork force — et l'EFF, La Quadrature du Net et l'OSI ont tous note que la formulation actuelle du reglement criminaliserait de fait toute messagerie chiffree non auditee par l'UE. La consequence pratique : ce n'est pas seulement Signal qui sort, c'est tout l'ecosysteme des messageries federees (Matrix, XMPP+OMEMO) qui devient juridiquement risque pour ses operateurs europeens.

Le chiffre : 311 contre 228 — la marge du rejet d'avril, etroite mais reelle. Le vote de juillet se jouera probablement dans le meme couloir.

📎 Sources : EFF — EU Parliament Blocks Mass-Scanning of Our Chats: What's Next? · State of Surveillance — Chat Control Is Dead. Long Live Chat Control.

TeamPCP : la nouvelle ecole d'attaque vise les humains, pas les depots

Depuis mars 2026, un groupe identifie sous le nom de TeamPCP a compromis successivement Trivy (le scanner de vulnerabilites open source maintenu par Aqua Security), KICS (analyse de code statique de Checkmarx), puis a essaime via npm en empoisonnant des paquets telecharges des dizaines de millions de fois par semaine. Le mode operatoire n'est pas un zero-day dans le code : c'est une chaine d'ingenierie sociale meticuleuse contre les mainteneurs. Dans le cas axios, l'attaquant s'est presente a Jason Saayman comme le fondateur d'une entreprise cherchant a sponsoriser le projet, l'a invite dans un faux Slack, puis sur un appel Teams a obtenu l'installation d'un « outil de diagnostic » qui a en realite exfiltre ses tokens de publication npm.

Ce n'est plus l'exception, c'est le motif. La sequence — flatterie, faux contexte professionnel, pression legere, demande d'action concrete — replique exactement le manuel utilise contre Lasse Collin avec xz-utils en 2024. Sauf que cette fois, l'operation a ete menee a l'echelle industrielle : plusieurs identites synthetiques actives en parallele, des semaines de pre-positionnement sur GitHub avec de vraies contributions techniques pour batir la credibilite, et des tokens revendus a des operateurs aval (un initial access broker du logiciel libre, en somme). Le rapport ReversingLabs date de cette semaine quantifie la derive : +73 % de paquets open source malveillants en 2025, avec une acceleration nette en 2026.

Pour le defenseur, le probleme est structurel. Tant que des projets critiques pour l'infrastructure mondiale dependent d'une seule personne epuisee, sans budget, sans support juridique, et sans formation a la securite operationnelle, l'angle d'attaque humain reste ouvert. La Software Freedom Conservancy et l'OpenSSF poussent pour des modeles de financement direct des mainteneurs, mais le tempo du financement reste ridiculement en retard sur le tempo de l'attaque.

Le chiffre : 454 600 paquets malveillants identifies sur les depots publics en 2025, contre 55 000 en 2022 — une multiplication par 8 en trois ans.

📎 Sources : Huntress — Tradecraft Tuesday Recap: axios npm Supply Chain Compromise · ReversingLabs — 2026 Software Supply Chain Security Report

Quasar Linux (QLNX) : le RAT qui aspire les credentials du developpeur

Trend Micro a publie cette semaine l'analyse complete d'un RAT (remote access trojan) Linux baptise Quasar Linux (QLNX), dont la signature operationnelle est sans ambiguite : il vise les machines de developpeurs et d'ingenieurs DevOps. La cible n'est pas la donnee utilisateur, c'est la cle du royaume : .npmrc (tokens npm), .pypirc (PyPI), .git-credentials, .aws/credentials, .kube/config, fichiers .env. Une fois ces fichiers exfiltres, l'attaquant a tout ce qu'il faut pour publier des paquets empoisonnes, deployer du code en production, ou acceder a l'infrastructure cloud cible.

C'est la jonction logique avec la story precedente. La ou TeamPCP attaque la psychologie du mainteneur, QLNX attaque sa machine. Combines, les deux vecteurs forment une chaine complete : compromettre le developpeur → publier le paquet → atteindre les utilisateurs aval → boucler par les credentials cloud. Le rootkit PAM integre et le module backdoor PAM signalent une qualite d'ingenierie qui n'est plus celle d'un script-kiddie — c'est de l'outillage d'acteur etatique ou para-etatique, vendu ou distribue dans des cercles fermes.

L'angle Liberte ici depasse la cybersecurite technique. Si le poste de travail du developpeur d'un protocole de privacy (Signal, Tor, Tails) tombe, ce n'est pas une fuite de donnees — c'est une victoire potentielle sur le chiffrement lui-meme, par le bas. La separation hygiene operationnelle / poste de travail principal, longtemps consideree comme la paranoia des cryptographes, redevient une exigence de base.

📎 Source : Trend Micro — Quasar Linux (QLNX): A Silent Foothold in the Software Supply Chain

Linux 7.1-rc2 publie

Torvalds a sorti la deuxieme release candidate de la 7.1 le 3 mai apres une fenetre de fusion calme. Le travail de queue pour la 7.2 commence deja : pull request initiale drm-misc-next envoyee, et la mainline 7.2 est attendue pour l'ete. Cote drivers, le RTL8159 entre en mainline — modeste, mais c'est ce genre d'increment qui rend Linux installable « out of the box » sur du materiel recent sans plonger dans des depots tiers. Pour le developpeur sur Lenovo Legion ou Framework, la 7.2 sera celle qui reduit encore la liste des composants necessitant des firmwares non-libres.

📎 Source : Phoronix — Linux 7.1-rc2 Released Following A Quiet Week

MetInfo (CMS open source) sous attaque active

CVE-2026-29014 a declenche une vague d'exploitations a partir du 1er mai, concentree sur les IP chinoises et hongkongaises. C'est le genre de news qu'on classerait normalement en cybersecurite — mais elle pose une question Liberte concrete : MetInfo est massivement utilise pour heberger des sites de petites entreprises et d'associations en Asie, dont des structures civiques locales. Une vague d'exploitations massives sur ce socle ouvre la porte a des defacages politiques cibles — pas seulement a du minage de cryptomonnaie.

La Menace du Jour

L'industrialisation de l'ingenierie sociale contre les mainteneurs

Le pattern qu'on appelait encore « affaire xz » en 2024 est devenu une categorie d'attaque a part entiere. Acteur : groupes professionnels (TeamPCP et probablement plusieurs autres non encore nommes). Mecanisme : pre-positionnement de longue duree comme contributeur, gain de confiance, escalade vers une demande d'installation ou d'acces, exfiltration de tokens. Victime : le mainteneur isole, financierement precaire, socialement expose sur des forges publiques. Resistance : OpenSSF (Alpha-Omega program), Software Freedom Conservancy, GitHub Security Lab — mais le delta entre attaquants finances et defenseurs benevoles continue de s'elargir.

Le point politique est rarement nomme. La societe de l'information dans son ensemble depend gratuitement du travail d'individus a qui elle ne paie ni assurance, ni temps de formation securite, ni avocat quand un Etat vient frapper. Tant que ce desequilibre persiste, chaque nouvelle attaque contre un mainteneur n'est pas un accident — c'est la consequence previsible d'un modele economique qui externalise la securite de toute l'infrastructure numerique sur des benevoles.

Signaux Faibles
  • Trellix confirme une fuite de code source partielle ; RansomHouse revendique le 7 mai. A surveiller — un editeur de XDR/EDR dont le code fuite est un risque structurel pour ses clients.
  • « 2026: The Year of AI-Assisted Attacks » — la lecture The Hacker News converge avec celle d'OpenSSF : la velocite d'attaque a bascule avec la generalisation des LLMs offensifs ; la velocite de defense, non.
  • Compromis GitHub Actions de Trivy / KICS — premier cas documente ou la chaine d'outils de securite elle-meme devient le vecteur. La meta-question : qui scanne le scanner ?
La Connexion

Trois histoires aujourd'hui, un seul fil. Chat Control 2.0 cherche a briser le chiffrement par la loi ; TeamPCP cherche a le briser par la psychologie ; QLNX cherche a le briser par la machine. Trois angles d'attaque sur le meme objet — l'autonomie cryptographique de l'individu — depuis trois directions differentes. Ce qui rend la defense difficile, c'est que la resistance a chacune releve d'institutions distinctes (Parlement europeen pour la premiere, OpenSSF/SFC pour la seconde, hygiene operationnelle individuelle pour la troisieme), qui ne parlent pas la meme langue et ne partagent pas le meme calendrier.

L'open source est le terrain commun. C'est lui qui rend Signal possible, lui qui rend les paquets npm utiles, lui qui rend Linux installable sans signer un EULA hostile. Quand on l'attaque, on attaque la condition materielle de la liberte numerique — pas seulement « du code ».

Trois Hypotheses pour la Semaine
  1. Surveiller les positions des Etats membres cles (Allemagne, France, Pays-Bas) avant le trilogue suivant — l'Allemagne pourrait basculer et modifier l'equilibre du vote de juillet sur Chat Control.
  2. Surveiller le rapport OpenSSF attendu sur la sante des projets single-maintainer : il pourrait servir de base a une proposition de financement structurel europeen via l'EU Sovereign Tech Fund.
  3. Surveiller un nouvel incident TeamPCP-like avant fin mai — le tempo observe depuis mars (un compromis majeur toutes les 4-6 semaines) suggere qu'un nouveau paquet structurant tombera avant le 1er juin.
