🔐 ENCRYPTION & BACKDOORS

"Crypto Wars 2026" — La Bataille du Chiffrement s'Intensifie

Les faits :

  • UK et gouvernements européens augmentent la pression sur le chiffrement E2E
  • Justification officielle : Détection CSAM (child abuse) et terrorisme
  • Signal a menacé de quitter le UK si backdoor imposée
  • Approche "subtile" : pas d'interdiction frontale, mais restrictions progressives

Source primaire : Computer Weekly, CDT

🔒 Source de référence : CDT (Center for Democracy & Technology) — expertise juridique et technique

Contexte historique :

  • 1990s : Crypto Wars 1.0 (Clipper Chip, export controls)
  • 2016 : FBI vs Apple (San Bernardino)
  • 2018 : Australie TOLA Act — premier pays à forcer backdoors légales
  • 2026 : Crypto Wars 2.0 — approche réglementaire vs frontale

Position des acteurs :

Ce que ça signifie concrètement :

  • Si adopté : Messages privés scannés par algorithmes
  • Faux positifs inévitables (taux d'erreur même faible = millions d'innocents flaggés)
  • Précédent pour élargir à d'autres contenus

Sources : Computer Weekly, CDT

Chat Control (CSAR) — Printemps 2026

Les faits :

  • Child Sexual Abuse Regulation (EU)
  • Adoption attendue printemps 2026
  • Proposition : Plateformes scannent "volontairement" messages privés
  • Critique : "Volontaire" = obligatoire de facto

Évaluation :

AspectRéalité
"Volontaire"Pression réglementaire rend obligatoire
EfficacitéHash matching contournable, IA génère faux positifs
PrécédentOuvre porte à surveillance généralisée

Citation Patrick Breyer (ex-MEP) :

"Warrantless and error-prone mass surveillance of EU citizens by US technology companies"

Source : Computer Weekly

Australie TOLA Act — Le Précédent Existant

Les faits :

  • Décembre 2018 : Premier pays à légaliser backdoors forcées
  • Système à 3 niveaux :
  1. Technical Assistance Requests (volontaire)
  2. Technical Assistance Notices (obligatoire + amendes)
  3. Technical Capability Notices (construire backdoor ou sanctions)

Pourquoi ça compte :

  • Modèle pour UK, EU, autres
  • Démontre que c'est légalement faisable
  • Pas d'effondrement catastrophique visible = argument pour expansion

🏛️ SURVEILLANCE ÉTATIQUE

US DHS — Data Brokers pour Tracking

Les faits :

  • 15 janvier 2026 : Alerte Minnesota Attorney General Ellison
  • ICE utilise outils avancés pour identifier et tracker personnes
  • Sources : Activité en ligne, apps, smart devices, véhicules
  • Accès via data brokers (pas de mandat nécessaire)

Source primaire : Minnesota AG Office

Ce que ça signifie concrètement :

  • Contournement du 4ème amendement via achat de données
  • Données collectées commercialement = accessibles au gouvernement
  • Pas de consentement, pas de notification

Recommandations AG Ellison :

  • Limiter partage données avec apps
  • Vérifier permissions devices
  • Utiliser VPN, messageries chiffrées

Source : Minnesota AG

California DROP — Outil de Suppression de Masse

Les faits :

  • Lancé 1er janvier 2026
  • Delete Request and Opt-out Platform
  • Une requête = suppression chez TOUS les data brokers enregistrés en Californie
  • Premier outil de ce type aux US

Impact :

  • Simplifie drastiquement l'exercice des droits
  • Pression sur data brokers
  • Modèle pour autres états ?

Source : IAPP
